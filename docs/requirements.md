# 项目开发需求

## 项目名称

DOCX 格式统一工具 (Table Format Unifier)

## 项目概述

一款运行在 Windows 上的桌面软件，帮助用户将格式不规范的 DOCX 文件，按照模板文件的格式标准，重新生成为格式统一、排布规范的 DOCX 文件。

## 用户场景

1. 用户有一份**输入文件**：内容基本正确，但格式不统一（字体混乱、表格列数不对、排版粗糙等）
2. 用户有一份**模板文件**：格式标准、排版规范，定义了期望的最终样式
3. 用户使用本软件，输入上述两个文件，一键生成**输出文件**：内容是输入文件的文字，格式与模板文件完全一致

## 功能需求

### F1 — 文件选择
- 用户可通过文件选择对话框选择输入文件（.docx）
- 用户可通过文件选择对话框选择模板文件（.docx）
- 用户可指定输出文件的保存路径

### F2 — 段落格式统一（正文部分）
- 保留模板文件的段落格式：字体、字号、加粗、斜体、颜色、对齐方式、行间距、段前间距、段后间距
- 将输入文件的文字内容填入模板对应的段落位置

### F3 — 表格格式统一
- 保留模板文件的表格结构：行数、列数、合并单元格、列宽、行高
- 保留模板文件的表格样式：边框、底色、文字格式
- 将输入文件表格中的文字内容填入模板表格的对应单元格

### F4 — 智能内容匹配
- 优先通过相同的"字段名/标签"进行匹配（如输入文件中有"项目名称：XXXX"，模板中也有"项目名称"字段）
- 无法通过字段名匹配的内容，按出现顺序依次对应
- 输入文件中无法匹配的多余内容，附加到输出文件末尾，保留其原格式

### F5 — 手动映射调整（高级功能）
- 用户可打开"手动调整映射"界面
- 查看和修改输入列与模板列的对应关系
- 默认关闭此功能，日常使用走自动匹配

### F6 — 处理日志与状态反馈
- 实时显示处理进度和状态信息
- 显示匹配结果摘要
- 转换完成后弹出成功提示

## 非功能需求

### N1 — 操作系统
- Windows 10 / Windows Server 2022 及以上

### N2 — 用户体验
- UI 简洁直观，不要求用户具备任何技术背景
- 文件选择使用常见的"浏览"按钮模式
- 转换操作一键完成

### N3 — 格式保真度
- 输出文件的字体、字号、对齐、行距、表格布局必须与模板文件一致
- 当输入文件的文字与模板完全相同时，输出文件应与模板文件逐字节一致（理想目标）

### N4 — 分发方式
- 最终应打包为单个 .exe 文件，用户无需安装 Python 或其他依赖

## 参考文件

项目 `参考文件/` 目录下有两份文件可用于开发和测试：
- `Reference Input.docx` — 输入文件示例（格式不规范）
- `Reference Template.docx` — 模板文件示例（格式标准）
