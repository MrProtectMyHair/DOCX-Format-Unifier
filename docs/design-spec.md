# UI / 交互设计规范

## 设计原则

- **简洁**：主窗口不超过 3 类操作区域
- **直观**：使用 Windows 用户熟悉的"输入→处理→输出"模式
- **容错**：关键操作前校验，错误时有明确提示

## 组件规格

### 文件选择行
- 标签：固定宽度 80px，右对齐
- 输入框：拉伸填充，只读，显示已选文件路径
- 浏览按钮：固定宽度 60px，点击弹出系统文件对话框
- 三个文件选择行上下间距 10px

### 状态预览区域
- 多行文本框，只读
- 高度约 150px，带滚动条
- 字体：等宽字体 10pt
- 背景：浅灰 (#F5F5F5)

### 按钮
- "开始转换"按钮：主按钮，蓝色背景 (#1E90FF)，白色文字，宽度 120px
- "手动调整映射"按钮：次按钮，灰色边框，宽度 130px
- 按钮高度统一 36px
- 未选择文件时"开始转换"按钮禁用（灰色）

## 交互流程

```
启动应用
   │
   ▼
选择输入文件 ──▶ 选择模板文件 ──▶ (可选)选择输出位置
   │                                      │
   │   ┌──────────────────────────────────┘
   │   │  (可选) 点击"手动调整映射"
   │   │     │
   │   │     ▼
   │   │  映射配置对话框
   │   │     │
   │   │     ▼
   │   │  确认 / 取消
   │   │
   ▼   ▼
[开始转换] 按钮可用
   │
   ▼
显示处理进度
   │
   ▼
处理完成 ──▶ 弹出提示"转换完成"
   │
   (用户选择是否打开输出文件)
```

## 手动映射对话框

```
┌────────────────────────────────────────┐
│  表格列映射设置                         │
│                                        │
│  模板列                输入列           │
│  ┌────────────────┐   ┌──────────────┐ │
│  │ 1. 申报类别    │──▶│ 2. 申报类别   │ │
│  │ 2. 姓名        │──▶│ 1. 姓名       │ │
│  │ 3. 学历        │──▶│ 3. 学历       │ │
│  │ 4. 培训目标    │──▶│ (未匹配)      │ │
│  │ 5. 实施方式    │──▶│ (未匹配)      │ │
│  └────────────────┘   └──────────────┘ │
│                                        │
│  [自动匹配]  [重置]       [确认] [取消] │
└────────────────────────────────────────┘
```

- 左右两列列表，显示映射关系
- 可点选修改对应关系
- "自动匹配"按钮恢复默认的算法匹配结果

## 颜色规范

| 用途 | 颜色值 | 说明 |
|------|--------|------|
| 主按钮 | #1E90FF | 蓝色，突出主要操作 |
| 主按钮悬停 | #1873CC | 深蓝 |
| 次按钮文字 | #555555 | 深灰 |
| 背景 | #FFFFFF | 白色 |
| 状态区背景 | #F5F5F5 | 浅灰 |
| 成功提示 | #28A745 | 绿色 |
| 错误提示 | #DC3545 | 红色 |

## 字体规范

- 界面字体：Microsoft YaHei (Windows 默认中文字体)
- 界面字号：12pt（正文）、11pt（标签）、10pt（日志）

## 当前实现状态

- **手动映射弹窗** (`gui/mapping_dialog.py`)：占位版本，自动匹配已能处理跨行列映射
