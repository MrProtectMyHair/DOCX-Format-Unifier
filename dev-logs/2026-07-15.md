# 开发日志 — 2026-07-15

## 完成事项

### 版本总结

v1.0 功能完整，支持 5 组参考文件（01-05）全部正确转换。以下是 2026-05-28 之后完成的主要工作：

### 表单式段落填充（下划线保留）
- 新增 `_is_blank_run()` 和 `_fill_form_paragraph()`：自动识别模板中的下划线/空白占位，从输入提取对应值填入，保留原 run 的 underline、font 等所有格式
- 修复 `_strip_label_if_template_has_none` 去空格比较误判 → 标签前缀不再被错误剥离
- 修复 `_fill_form_paragraph` 锚点空格不一致导致匹配失败 → 增加正则空格模糊回退
- 修复无 blank run 段落降级时破坏模板 run 结构 → 去空格文本一致时保留模板
- 修复尾部截断空格误判 → `trailing.lstrip()` 后再搜索
- 修复未匹配段落的 label 匹配合并（如电话独立段落合并回 form 段落）

### 多表格支持
- `match_tables` 改为返回 `table_matches` 结构（每对 table 独立匹配）
- `_replace_table_cells` 和 `_fill_empty_cells_by_label` 改为接收单表参数，遍历所有表
- GUI 和 MappingDialog 适配

### 标签领地填充增强
- 新增 `_is_summary_label()`：检测"合计/总计/小计"时领地仅填紧邻一格
- `_strip_label_if_template_has_none` 增加下划线核心标签归一化比对

### 文件保存稳定性
- 临时文件路径从系统 Temp 改为输出目录（避免杀软扫描锁文件）
- 路径生成从 `mkstemp` 改为 `uuid`（不预创建文件）
- `copy2` 后强制 `os.chmod` 移除只读属性

### 文档更新
- 更新 `CLAUDE.md` 精简规则
- 更新 `docs/tech-spec.md` 同步最新算法
- 更新 `docs/execution-plan.md` 标注最终完成状态
- 新增 `README.md` 供 GitHub 展示
